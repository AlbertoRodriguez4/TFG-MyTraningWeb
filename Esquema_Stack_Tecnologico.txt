================================================================================
2. BACKEND: .NET 8 (API RESTful)
================================================================================

2.1 Stack Base
------------------------------------------------------------------------------
- Framework:        .NET 8.0 (LTS)
- Tipo de proyecto: ASP.NET Core Web API
- Lenguaje:         C# 12 (Nullable enabled, ImplicitUsings)
- Runtime:          Kestrel (puerto 6873)
- Formato de datos: JSON (System.Text.Json)
- Documentacion:    Swagger / OpenAPI (Swashbuckle.AspNetCore 6.6.2)


2.2 Capas de Arquitectura
------------------------------------------------------------------------------

   +----------------------------------------------------------+
   |  CAPA DE PRESENTACION  (Controllers / API Endpoints)     |
   |  Ej: CoachAIController, AuthController, MapController    |
   +----------------------------------------------------------+
                              |
                              v
   +----------------------------------------------------------+
   |  CAPA DE NEGOCIO       (Services / Logica de aplicacion) |
   |  Ej: CoachAIService, ExerciseService, EmailService       |
   +----------------------------------------------------------+
                              |
                              v
   +----------------------------------------------------------+
   |  CAPA DE ACCESO A DATOS (Repositories / EF Core)         |
   |  Ej: UserRepository, ExerciseRepository                  |
   +----------------------------------------------------------+
                              |
                              v
   +----------------------------------------------------------+
   |  CAPA DE PERSISTENCIA   (PostgreSQL via Npgsql + EF)     |
   +----------------------------------------------------------+


2.3 Dependencias Principales (NuGet)
------------------------------------------------------------------------------
| Paquete                                  | Version | Funcion                |
|------------------------------------------|---------|------------------------|
| Microsoft.EntityFrameworkCore            | 9.0.7   | ORM principal          |
| Npgsql.EntityFrameworkCore.PostgreSQL    | 9.0.4   | Driver PostgreSQL      |
| Microsoft.AspNetCore.Authentication.JwtBearer | 8.0.4 | Auth JWT          |
| System.IdentityModel.Tokens.Jwt          | 8.8.0   | Generacion de tokens   |
| BCrypt.Net-Next                          | 4.0.3   | Hash de contrasenas    |
| Swashbuckle.AspNetCore                   | 6.6.2   | Documentacion Swagger  |
| Microsoft.AspNetCore.OpenApi             | 8.0.11  | Soporte OpenAPI nativo |


2.4 Seguridad y Autenticacion
------------------------------------------------------------------------------
- Autenticacion:    JWT Bearer Tokens (asimetrico con clave simetrica)
- Claims:           Issuer, Audience, Role, NameIdentifier
- Proteccion:       ValidateLifetime, ValidateIssuerSigningKey
- Passwords:        Hasheadas con BCrypt (sal automatica)
- CORS:             Politica restringida al origen del frontend


2.5 Servicios en Segundo Plano (Background Services)
------------------------------------------------------------------------------
- NotificationBackgroundService: Hilo cron que se ejecuta diariamente a las
  08:00 UTC para comprobar inactividad de usuarios y suscripciones proximas
  a expirar, enviando emails automaticos.

3.3 Estructura de Carpetas del Frontend
------------------------------------------------------------------------------
 src/
 |-- Views/           # Paginas completas (HomeView, ChatView, ProfileView)
 |-- Components/      # Componentes reutilizables (ChatSidebar, PopUps...)
 |-- Stores/          # Pinia Stores (userStore, mapStore...)
 |-- JWT/             # Logica de autenticacion (tokens, interceptores)
 |-- i18n/            # Ficheros de traduccion (es.json, en.json, fr.json)
 |-- Styles/          # Hojas de estilo globales y temas
 |-- Router/          # Configuracion de rutas
 |-- Assets/          # Imagenes, fuentes, recursos estaticos

================================================================================
4. BASE DE DATOS: POSTGRESQL
================================================================================

4.1 Motor y Acceso
------------------------------------------------------------------------------
- Motor:            PostgreSQL
- Acceso:           Entity Framework Core 9 (Code First)
- Driver .NET:      Npgsql 9.0.3
- Migraciones:      Gestionadas via EF Migrations (aplicadas automaticamente
                    al iniciar el backend con db.Database.Migrate())


4.2 Esquema Relacional (Entidades principales)
------------------------------------------------------------------------------
   [Users] 1----* [Rooms] (propietario)
   [Users] 1----* [UserRooms] *----1 [Rooms] (participacion)
   [Users] 1----* [Tasks]
   [Users] 1----* [BodyMetrics]
   [Users] 1----* [Purchases] *----1 [Items]
   [Users] 1----1 [Subscriptions]
   [Users] 1----* [Achievements]
   [Rooms] 1----* [ChatMessages]
   [Exercises] *----* [Tasks] (via TaskExercise)

6. FLUJO DE UNA PETICION TIPICA (EJEMPLO: LOGIN + DATOS)
================================================================================

   USUARIO
      |
      v
   +---------------------------+
   | 1. Introduce credenciales |
   |    (Frontend Vue 3)       |
   +---------------------------+
      |
      | POST /api/auth/login
      | {email, password}
      v
   +---------------------------+
   | 2. AuthController recibe  |
   |    la peticion (C#)       |
   +---------------------------+
      |
      v
   +---------------------------+
   | 3. AuthService valida     |
   |    contra UserRepository  |
   |    (BCrypt.Verify)        |
   +---------------------------+
      |
      v
   +---------------------------+
   | 4. Genera JWT (JwtBearer) |
   |    con claims de rol/id   |
   +---------------------------+
      |
      | Respuesta: {token, user}
      v

   | 5. Frontend guarda token  |
   |    en Pinia / localStorage|
   +---------------------------+

      | GET /api/users/me
      | Authorization: Bearer <token>
      v
   +---------------------------+
   | 6. Middleware JWT valida  |
   |    firma, issuer y exp.   |
   +---------------------------+
