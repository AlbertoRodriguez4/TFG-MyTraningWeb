=============================================================
RESUMEN DEL STACK TECNOLOGICO - THE TRAINING HUB
=============================================================
Proyecto: TFG - MyTrainingWeb
Autor: Alberto Rodriguez Penalva
Estructura: Backend (C#/.NET) + Frontend (Vue 3) + Infraestructura
=============================================================

Este documento resume el stack tecnologico completo del proyecto,
organizado por capas para facilitar la creacion de un esquema visual.

=============================================================
1. INFRAESTRUCTURA / DEVOPS
=============================================================

[Contenedores]
  - Docker
  - Docker Compose

[Orquestacion]
  - Kubernetes (manifests YAML para API, Frontend, PostgreSQL, pgAdmin)

[Servidor Web / Proxy]
  - Nginx (servidor del frontend en produccion)

[CI/CD]
  - GitHub Actions (.github/workflows/docker-image.yml)

[Control de Versiones]
  - Git + GitHub

[Base de Datos]
  - PostgreSQL (puerto 3786 mapeado a 5432)
  - pgAdmin 4 (gestion grafica, puerto 80)

=============================================================
2. BACKEND (C# / .NET 8)
=============================================================

[Lenguaje y Framework]
  - C# 12
  - .NET 8.0 (ASP.NET Core Web API)
  - Nullable reference types + Implicit usings habilitados

[Arquitectura]
  - Controller -> Service -> Repository -> Entity Framework -> DB
  - Inyeccion de dependencias (Scoped)
  - Middleware personalizado: ExceptionHandlingMiddleware

[ORM y Base de Datos]
  - Entity Framework Core 9.0.7
  - Npgsql 9.0.3 (driver PostgreSQL para .NET)
  - Npgsql.EntityFrameworkCore.PostgreSQL 9.0.4
  - Migraciones EF Core (Migrations/)

[Autenticacion y Seguridad]
  - JWT Bearer Authentication (Microsoft.AspNetCore.Authentication.JwtBearer)
  - System.IdentityModel.Tokens.Jwt
  - BCrypt.Net-Next 4.0.3 (hashing de contrasenas)
  - Claims-based authorization

[Documentacion API]
  - Swagger / OpenAPI (Swashbuckle.AspNetCore 6.6.2)
  - Microsoft.AspNetCore.OpenApi 8.0.11

[Testing]
  - xUnit (Backend.Tests/Backend.Tests.csproj)

[Servicios Externos / APIs de Terceros]
  - Groq API (IA Coach AI)
      - Modelo: llama-3.3-70b-versatile
      - Timeout: 60 segundos
  - ExerciseDB API (base de datos de ejercicios)
      - Timeout: 30 segundos
  - Nominatim / OpenStreetMap (geocodificacion)
  - SMTP Gmail (envio de correos electronicos)

[Controladores Principales]
  - AuthController
  - UserController / UserRoomController
  - RoomController
  - PlanController / TaskController
  - ExerciseController
  - ItemController / PurchaseController
  - SubscriptionController
  - BodyMetricController
  - AchievementController
  - CoachAIController
  - GeocodificacionController
  - NotificationPreferenceController

[Middleware]
  - ExceptionHandlingMiddleware (manejo global de excepciones)

[Configuracion]
  - appsettings.json (connection strings, JWT, Email, AI, ExerciseDB)
  - appsettings.Development.json

=============================================================
3. FRONTEND (Vue 3 / TypeScript)
=============================================================

[Lenguaje y Framework]
  - TypeScript 5.8
  - Vue 3.5.13 (Composition API)

[Build Tool]
  - Vite 6.2.3
  - Vue TSC (type-checking)

[Estado Global]
  - Pinia 3.0.1

[Routing]
  - Vue Router 4.5.0

[UI Framework]
  - Vuetify 3.10.5 (Material Design)
  - Material Design Icons (@mdi/font 7.4.47)
  - Sass/SCSS 1.88.0

[Internacionalizacion]
  - Vue I18n 11.1.3
  - Idiomas: es.json, en.json, fr.json

[Librerias y Utilidades]
  - Chart.js 4.5.1 (graficos y estadisticas)
  - Leaflet 1.9.4 (mapas interactivos)
  - jwt-decode 4.0.0 (decodificacion de tokens JWT)
  - CORS (cors 2.8.5)

[Formateo y Calidad]
  - Prettier 3.5.3
  - npm-run-all2

[DevTools]
  - vite-plugin-vue-devtools 7.7.2
  - vite-plugin-vue-inspector

[Estructura de Carpetas]
  - src/views/     (vistas/paginas por modulo: auth, dashboard, plans, profile, rooms...)
  - src/components/ (componentes reutilizables)
  - src/stores/    (Pinia stores: userStore, exerciseStore, PlanStore, RoomStore...)
  - src/router/    (configuracion de rutas)
  - src/i18n/      (traducciones)
  - src/jwt/       (gestion de tokens)
  - src/styles/    (estilos globales)
  - src/utils/     (utilidades: logger.ts, api.ts)
  - src/composables/ (composables Vue: useAuthValidation, useTheme)

=============================================================
4. COMUNICACION FRONTEND <-> BACKEND
=============================================================

[Protocolo]
  - HTTP/REST (JSON)

[CORS]
  - Origenes permitidos: http://localhost:5173, http://127.0.0.1:5173

[Puertos]
  - Backend API:  http://127.0.0.1:6873
  - Frontend Dev: http://localhost:5173
  - Frontend Prod (Nginx): puerto 80

[Autenticacion]
  - JWT Bearer Token en header Authorization
